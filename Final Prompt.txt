   1.Missing settlements
   2.Late settlements
   3.Duplicate entries
   4.Rounding differences
   5.Invalid refunds

4. Keep implementation simple:
   1.use only lists and dictionaries
   2.small, clear functions

   print detected issues clearly in the console
   example:
     Missing settlements detected: [...]
     Late settlements detected: [...]etc.


6. At the end, call all functions and display results.

7. also include:
   simple test cases validating each issue

     1.what problem this solves
